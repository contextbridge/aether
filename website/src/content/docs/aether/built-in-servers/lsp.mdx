The `lsp` server provides code intelligence powered by Language Server Protocol daemons: symbol lookup, diagnostics, document structure, workspace search, and rename.

## Configuration

```json title=".aether/mcp.json"
{
  "servers": {
    "lsp": {
      "type": "in-memory"
    }
  }
}
```

In Aether's current built-in registration, the `lsp` server is constructed with the project root; no args are read from the MCP config. The standalone `LspMcp::from_args` code path also accepts `--root-dir <path>` for embedding use cases.

## Tools

| Tool                   | Description                                                                                         |
| ---------------------- | --------------------------------------------------------------------------------------------------- |
| `lsp_symbol`           | Go to definition, find references, hover info, implementations, incoming calls, and outgoing calls. |
| `lsp_document`         | List all symbols in a file.                                                                         |
| `lsp_check_errors`     | Get compiler errors and warnings without running a build command.                                   |
| `lsp_workspace_search` | Search for symbols across the workspace by name.                                                    |
| `lsp_rename`           | Rename a symbol across the workspace.                                                               |

## Path expectations

LSP tools work best with absolute file paths under the project root advertised in the server instructions. Use `lsp_document` before reading very large files when you only need the symbol outline.
