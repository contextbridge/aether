Aether is an open-source AI coding agent that runs in a terminal UI, as an ACP server for editors, or headlessly for automation. Agents are configured from project-local files: prompts, model choices, MCP tools, user-facing modes, and sub-agents all live under your project instead of being hardcoded into the binary.

### Design principles

- **Explicit prompts** — Agents start from authored prompt sources that you control.
- **Project-local modes** — Define user-invocable agents and agent-invocable sub-agents in `.aether/settings.json`.
- **MCP for tools** — Built-in and external MCP servers provide file operations, LSP, skills, tasks, surveys, plan review, and anything else you wire in.
- **Multiple surfaces** — Use the same agents in the TUI, editor integrations over ACP, or `aether headless` scripts.

## Getting started

<Steps>

1. **Install**

   Use one of the supported install paths:

   ```bash title="Homebrew"
   brew install contextbridge/tap/aether
   ```

   ```bash title="Shell installer"
   curl --proto '=https' --tlsv1.2 -LsSf https://github.com/contextbridge/aether/releases/latest/download/aether-agent-cli-installer.sh | sh
   ```

   ```bash title="Cargo"
   cargo install aether-agent-cli
   ```

   Cargo installs require **Rust 1.85+**. Install Rust with [rustup](https://rustup.rs) if you do not already have it.

   To build from source:

   ```bash
   git clone https://github.com/contextbridge/aether.git
   cd aether
   cargo build --release
   ```

   The main CLI is written to `target/release/aether`. The workspace build also produces the lower-level standalone `wisp` TUI binary at `target/release/wisp`.

2. **Create your first agent**

   The explicit setup command is:

   ```bash
   cd your-project
   aether agent new .
   ```

   Running plain `aether` in a directory without `.aether/settings.json` does the same onboarding first, then launches the terminal UI backed by `aether acp`.

   The setup wizard walks you through choosing an agent name, model, prompt files, and built-in MCP tools. For an initial agent named `Build`, the generated files look like this:

   ```text
   .aether/
     settings.json  # Agent definitions
     mcp.json        # MCP server config
     BUILD.md        # Agent prompt scaffold; filename follows the agent name
   AGENTS.md         # Optional project instructions when selected by the wizard
   ```

   Adding another agent to an existing project creates agent-local files instead, for example `.aether/agents/researcher/RESEARCHER.md` and `.aether/agents/researcher/mcp.json` when that agent has its own MCP selection.

   Built-in in-memory servers you can enable include `coding`, `skills`, `tasks`, `subagents`, `survey`, and `plan`. LSP-backed code intelligence is included in `coding` unless you disable it.

   :::note[Set up your LLM credentials]
   Most cloud providers need an API key, for example `export ANTHROPIC_API_KEY=sk-ant-...` or `export OPENAI_API_KEY=sk-...`. Codex uses an OAuth flow triggered from the client, while local providers such as Ollama and llama.cpp are discovered from their running local servers.
   See [LLM Providers](/aether/configuration/llm-providers/) for model syntax and availability.
   :::

3. **Run it**

   **Interactive TUI**

   ```bash
   aether
   ```

   The default experience: a terminal interface for conversations with your selected user-invocable agent.

   **IDE / editor (via ACP)**

   ```bash
   aether acp
   ```

   Exposes agents over [ACP](https://agentclientprotocol.com/get-started/introduction) for [editor integration](/aether/running/editor-integration/).

   **Headless**

   ```bash
   aether headless "Explain the main function in this project"
   ```

   Run a single prompt and exit — ideal for scripts and CI. See [Headless CLI](/aether/running/headless/) for all flags and output formats.

</Steps>

## Core commands

| Command                                 | Purpose                                                                      |
| --------------------------------------- | ---------------------------------------------------------------------------- |
| `aether`                                | Run onboarding if needed, then launch the terminal UI backed by `aether acp` |
| `aether agent new [path]`               | Create an initial project config or add another agent                        |
| `aether agent list [path]`              | List configured agents for a project                                         |
| `aether agent remove <name> [path]`     | Remove an agent from a project                                               |
| `aether headless [OPTIONS] [PROMPT]...` | Run one prompt and exit                                                      |
| `aether acp [OPTIONS]`                  | Start the ACP server for editors or Wisp                                     |
| `aether show-prompt [OPTIONS]`          | Print the assembled prompt and tool summary for debugging                    |

A top-level `--sandbox-image <image>` flag is available on the CLI when you want to run the command inside a Docker sandbox.
