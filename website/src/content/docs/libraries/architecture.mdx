Aether is organized as a Cargo workspace with focused, single-responsibility packages. Each package can be used independently or composed together.

## Package map

| Package            | Crate            | Purpose                                                                                                                    |
| ------------------ | ---------------- | -------------------------------------------------------------------------------------------------------------------------- |
| **aether-core**    | `aether_core`    | Agent runtime ‚Äî conversation loop, context management, tool dispatch, streaming                                            |
| **llm**            | `llm`            | Unified LLM provider interface ‚Äî Anthropic, OpenRouter, OpenAI, DeepSeek, Gemini, Ollama, Bedrock, and more                |
| **mcp-servers**    | `mcp_servers`    | Built-in MCP tool servers ‚Äî coding (files, bash, search, LSP), standalone LSP, skills, tasks, sub-agents, survey, and plan |
| **aether-cli**     | `aether_cli`     | CLI binaries ‚Äî headless mode, ACP server, init wizard, show-prompt                                                         |
| **aether-project** | `aether_project` | Project configuration ‚Äî loads and validates `settings.json`, resolves MCP configs and prompt globs                         |
| **wisp**           | `wisp`           | Interactive TUI ‚Äî markdown rendering, syntax highlighting, inline diffs, command picker                                    |
| **crucible**       | `crucible`       | Evaluation framework ‚Äî assertion-based testing of agent behavior                                                           |
| **mcp-utils**      | `mcp_utils`      | MCP client utilities ‚Äî connecting to and managing external MCP servers                                                     |
| **tui**            | `tui`            | Low-level TUI rendering primitives used by Wisp                                                                            |
| **acp-utils**      | `acp_utils`      | Agent Client Protocol utilities ‚Äî client and server helpers                                                                |
| **aether-lspd**    | `aether_lspd`    | LSP server daemon ‚Äî standalone language server integration                                                                 |
| **llm-codegen**    | `llm_codegen`    | Code generation for LLM provider model catalogs                                                                            |
| **utils**          | `utils`          | Shared utilities used across packages                                                                                      |

## Key design decisions

- **MCP for all tools** ‚Äî There is no built-in tool system. Every tool comes through MCP, whether from in-memory servers or external processes. This keeps the core runtime minimal and tools composable.
- **Provider abstraction** ‚Äî The `llm` crate defines a unified interface. Adding a new provider means implementing a single trait, not touching the agent runtime.
- **Config as data** ‚Äî Agent definitions, tool access, and prompt composition are all declarative JSON/markdown. No code changes needed to reconfigure agents.
- **Workspace isolation** ‚Äî Each package has a focused responsibility. You can use `llm` without `mcp-servers`, or `aether-core` without `wisp`.
