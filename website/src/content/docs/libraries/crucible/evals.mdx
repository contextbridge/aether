Crucible is Aether's evaluation framework. It runs a prompt against an implementation of the `Agent` trait, captures the agent's messages, and returns an `EvalReport` you can assert against.

```toml title="Cargo.toml"
[dependencies]
crucible = { path = "../packages/crucible" }
```

## Running an eval

The main entry point is `run_eval(agent, prompt, workspace)`:

```rust
use crucible::{FakeAgent, Workspace, assert_tool_called, run_eval};

#[tokio::test]
async fn calls_bash() {
    let agent = FakeAgent::with_tool_call("bash", "ok");
    let report = run_eval(&agent, "inspect the repository", Workspace::empty().unwrap())
        .await
        .unwrap();

    assert_tool_called(&report, "bash");
}
```

`run_eval` returns an `EvalReport` with the prompt, isolated workspace, captured `AgentEvalMessage` values, and git diff metadata when the workspace came from a git repo.

## Workspaces

Each eval runs in an isolated temporary workspace:

```rust
use crucible::{GitRepoSpec, Workspace};

// Empty temp directory
let workspace = Workspace::empty()?;

// Copy the contents of a local fixture directory
let workspace = Workspace::from_dir("./test-fixtures/my-project")?;

// Clone a repo, check out a starting commit, and optionally run in a subdir
let workspace = Workspace::from_git_repo(GitRepoSpec {
    url: "https://github.com/user/repo".into(),
    start_commit: "start-commit-sha".into(),
    gold_commit: "gold-commit-sha".into(),
    subdir: Some("packages/app".into()),
})?;
```

For git workspaces, `EvalReport::agent_diff()` captures the agent's unstaged changes and `EvalReport::reference_diff()` captures the diff between `start_commit` and `gold_commit`.

## Assertions

Crucible ships assertion helpers for common report checks:

| Helper                                             | What it checks                            |
| -------------------------------------------------- | ----------------------------------------- |
| `assert_tool_called(&report, name)`                | Tool was called at least once             |
| `assert_tool_call_count(&report, name, count)`     | Tool was called exactly `count` times     |
| `assert_tool_call_with_args(&report, name, &json)` | Tool was called with exact JSON arguments |

You can also assert directly against the public `EvalReport` API:

```rust
assert_eq!(report.prompt(), "inspect the repository");
assert!(report.workspace().path().exists());
assert!(report.tool_called("bash"));
assert_eq!(report.tool_call_count("bash"), 1);

for call in report.tool_calls("bash") {
    let args = call.arguments_json()?;
    assert_eq!(call.name, "bash");
}
```

When an assertion fails, the helpers include `report.failure_context()` with the workspace path, prompt, diff summary, and captured agent messages.

## FakeAgent

Use `FakeAgent` for deterministic tests of your eval harness:

```rust
use crucible::{AgentEvalMessage, FakeAgent};

let agent = FakeAgent::new(vec![
    AgentEvalMessage::AgentText("I'll create the file".into()),
    AgentEvalMessage::ToolCall {
        name: "write_file".into(),
        arguments: r#"{"filePath":"hello.rs"}"#.into(),
    },
    AgentEvalMessage::Done,
]);

let agent = FakeAgent::writes_file("hello.rs", "fn main() {}\n");
```

`FakeAgent::writes_file` writes into the eval workspace before sending its messages, so you can assert on real file state without invoking an LLM.

## AetherAgent

`AetherAgent` adapts the real Aether runtime to the `Agent` trait:

```rust
use crucible::{AetherAgent, Prompt};
use futures::FutureExt;
use mcp_servers::CodingMcp;

let llm = /* any StreamingModelProvider + Clone */;
let agent = AetherAgent::new(llm)
    .with_system_prompt(Prompt::text("You are a careful coding agent."))
    .with_mcp_server_factory("coding", Box::new(|_args, _input| {
        async move { CodingMcp::new().into_dyn() }.boxed()
    }));
```

Registered in-memory MCP factories are exposed to the agent automatically. You can also add external MCP servers with `.with_mcp_json(".aether/mcp.json")`.

## LLM judge

`EvalReport::judge` runs a second LLM over the report and expects a JSON metric response:

```rust
let judgment = report
    .judge(&judge_llm, |ctx| {
        format!(
            "Did the agent satisfy the prompt?\n\nPrompt:\n{}\n\nMessages:\n{:?}",
            ctx.original_prompt,
            ctx.messages
        )
    })
    .await?;

assert!(judgment.passed(), "{}", judgment.reason());
```

The judge response must deserialize into an eval metric, for example:

```json
{ "type": "binary", "success": true, "reason": "implemented correctly" }
```
