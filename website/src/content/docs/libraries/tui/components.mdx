The `tui` crate provides a small component-based terminal UI framework. Interactive widgets implement `Component`: they receive terminal events and render a `Frame`.

```toml title="Cargo.toml"
[dependencies]
tui = { package = "aether-tui", version = "0.2.5" }
```

## Component trait

```rust
pub trait Component {
    type Message;

    fn on_event(&mut self, event: &Event) -> impl Future<Output = Option<Vec<Self::Message>>>;
    fn render(&mut self, ctx: &ViewContext) -> Frame;
}
```

| Method            | Description                                                                                           |
| ----------------- | ----------------------------------------------------------------------------------------------------- |
| `on_event(event)` | Handle input, returning `None` if unhandled, `Some(vec![])` if consumed, or messages to bubble upward |
| `render(ctx)`     | Render the component into a `Frame`                                                                   |

`Message` is an associated type — each component defines its own message enum for communicating state changes upward.

## Events

```rust
pub enum Event {
    Key(KeyEvent),
    Paste(String),
    Mouse(MouseEvent),
    Tick,
    Resize(Size),
}
```

`Event::try_from(crossterm::event::Event)` filters key releases and converts key presses, paste, mouse, and resize events into the framework event type.

## Built-in widgets

| Widget              | Feature  | Description                                                    |
| ------------------- | -------- | -------------------------------------------------------------- |
| `TextField`         | —        | Single-line text input                                         |
| `SelectList`        | —        | Scrollable selection list over items implementing `SelectItem` |
| `RadioSelect`       | —        | Single-choice selection                                        |
| `MultiSelect`       | —        | Multi-choice selection                                         |
| `Checkbox`          | —        | Boolean toggle                                                 |
| `NumberField`       | —        | Numeric input                                                  |
| `Form`              | —        | Multi-field form container                                     |
| `Spinner`           | —        | Loading indicator                                              |
| `Panel`             | —        | Bordered container                                             |
| `BorderedTextField` | —        | Single-line input with labeled border                          |
| `Stepper`           | —        | Horizontal multi-step progress indicator                       |

### TextField

```rust
use tui::TextField;

let mut field = TextField::new("initial value".to_string());
field.insert_str_at_cursor(" + more");
field.set_content_width(40);
```

### SelectList

```rust
use tui::{Line, SelectItem, SelectList, ViewContext};

struct Choice {
    label: String,
}

impl SelectItem for Choice {
    fn render_item(&self, selected: bool, ctx: &ViewContext) -> Line {
        if selected {
            ctx.theme.selected_row_line(self.label.clone())
        } else {
            Line::new(self.label.clone())
        }
    }
}

let items = vec![Choice { label: "Option One".into() }];
let mut list = SelectList::new(items, "no options");
```

### BorderedTextField

A single-line text input rendered inside a box with the label intersecting the top border:

```rust
use tui::BorderedTextField;

let mut field = BorderedTextField::new("Name", "my-agent".into());
field.set_width(40);
```

### Stepper

A horizontal multi-step progress indicator showing complete, current, and upcoming steps. It renders a single `Line`, not a full `Component`:

```rust
use tui::{StepVisualState, Stepper, StepperItem};

let items = vec![
    StepperItem { label: "Configure", state: StepVisualState::Complete },
    StepperItem { label: "Install", state: StepVisualState::Current },
    StepperItem { label: "Finish", state: StepVisualState::Upcoming },
];
let stepper = Stepper { items: &items, separator: " > ", leading_padding: 2 };
let line = stepper.render(&ctx);
```

### SplitPanel

A two-panel layout with focus managed by `Tab` / `BackTab`:

```rust
use tui::{SplitLayout, SplitPanel};

let layout = SplitLayout::fraction(1, 3, 20, 60); // 1/3 of width, clamped [20, 60]
let split = SplitPanel::new(left_component, right_component, layout)
    .with_separator("│", Default::default())
    .with_resize_keys();
```

### Gallery

A browseable list of named components with a live preview pane (built on `SplitPanel`):

```rust
use tui::Gallery;

let entries = vec![
    ("TextField".into(), text_field_demo),
    ("Checkbox".into(), checkbox_demo),
];
let gallery = Gallery::new(entries);
```

### Combobox (feature: `picker`)

A searchable dropdown with fuzzy matching:

```rust
use tui::{Combobox, Searchable};

#[derive(Clone)]
struct MyItem {
    name: String,
}

impl Searchable for MyItem {
    fn search_text(&self) -> String {
        self.name.clone()
    }
}

let items = vec![MyItem { name: "foo".into() }];
let mut picker = Combobox::new(items);
```

## FocusRing

Manages focus across multiple components:

```rust
use tui::FocusRing;

let mut focus = FocusRing::new(3); // 3 focusable elements
focus.focus_next();
focus.focus_prev();
let current = focus.focused();
```

## Composing components

Components compose by embedding child components and delegating events:

```rust
use tui::{Component, Event, Frame, TextField, ViewContext};

struct MyApp {
    input: TextField,
}

impl Component for MyApp {
    type Message = ();

    async fn on_event(&mut self, event: &Event) -> Option<Vec<Self::Message>> {
        self.input.on_event(event).await.map(|_| vec![])
    }

    fn render(&mut self, ctx: &ViewContext) -> Frame {
        self.input.render(ctx)
    }
}
```
